Sending Crash Reports (macOS)

On this page

If Toku Budget crashes on your Mac, sending the crash report helps us fix it fast. This page shows how to find the crash log and email it to us.

What to send

  • The crash file (ends with .crash).
  • Your app version (Toku Budget → About).
  • macOS version ( → About This Mac).
  • Steps to reproduce (what you clicked or imported right before it crashed).

Email: grantmarcus1994@gmail.com.

Method 1 — Console app (quick)

  1. Open Console (spotlight search for “Console”).
  2. In the sidebar, under Reports, click User Reports (or Crash Reports on some macOS versions).
  3. Use the search box to filter for Toku or Toku Budget.
  4. Select the most recent entry whose name starts with Toku_Budget and ends with .crash.
  5. Right-click the report and choose Reveal in Finder, or click Share → Save….
  6. Attach that .crash file to your email.

Method 2 — Finder (direct path)

The system stores user crash logs here:

  • ~/Library/Logs/DiagnosticReports/ (most common)
  • ~/Library/Logs/CrashReporter/ (older macOS)
  1. In Finder, press Shift + ⌘ + G and paste one of the paths above.
  2. Look for files named like Toku_Budget_YYYY-MM-DD-....crash.
  3. Pick the newest file after the crash and attach it to your email.

Optional — Capture live logs

If the crash is repeatable, a short live log can help:

  1. Open Console and select your Mac under Devices.
  2. Use the search box and type Toku to filter.
  3. Click Start (or just reproduce the issue if the stream is already live).
  4. Reproduce the crash, then choose File → Save… to export a log snippet.
  5. Attach the saved log along with the crash file.
No special steps for symbolication: Send the raw .crash file. We’ll symbolicate it with our build info.

Where else to look (Analytics Data)

On some systems, crash diagnostics also appear under:

  1. Open System Settings → Privacy & Security → Analytics & Improvements.
  2. Click Analytics Data and look for entries with Toku_Budget.
  3. Select the item, click Share → Save, and attach it to your email.

Email template

Subject: Toku Budget Crash Report

App version: 1.x (build ####)
macOS version: 14.x (e.g., Sonoma)
What I was doing: [brief steps]

Attached: Toku_Budget_YYYY-MM-DD-....crash
(Optional) Attached: live console log

Thank you

Crash reports are priceless for fixing issues quickly. We appreciate the help! If you prefer, you can also just drag the .crash file onto an email to grantmarcus1994@gmail.com.

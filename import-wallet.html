Import from Wallet

On this page

Direct “Import from Wallet” isn’t available yet. We’re designing a simple, private flow to bring in transactions from popular wallets and banking apps. Until then, you can export from your wallet and use Import CSV.

Status

Coming soon. The first release will focus on a clean CSV/statement handoff and one-click mapping, with per-transaction currency preserved. JPY UI display is planned right after (multi-currency docs here).

What “Import from Wallet” will do

  • Accept statements shared from wallet/banking apps (CSV preferred; OFX/QIF if provided).
  • Auto-detect headers (Date, Amount or Debit/Credit, Description/Note, Category, Currency).
  • Respect your chosen Amount Mode: Debit/Credit columns or Single Amount (negatives = expenses).
  • Preserve the original Currency code on each transaction.
  • Skip duplicates using a content hash (date + amount + kind + currency + note).
  • Let you save a mapping as a reusable Template that auto-applies next time.

Privacy

Toku Budget is local-first. Your data is stored on your device (Core Data). If you enable Premium iCloud sync, your encrypted data syncs via your Apple ID. We don’t see or sell your data.

Workaround you can use today

  1. Export from your wallet/app. Look for Export, Download statement, or Share. Choose CSV if available.
  2. Open Toku Budget → Import CSV. (macOS: Toolbar → Import/Export → Import CSV… or Settings → Import.)
  3. Map your columns. At minimum you need Date and either Amount or Debit/Credit. Set the correct Date Format and Delimiter. If your file lacks a currency column, set Currency Fallback (defaults to USD).
  4. Pick Amount Mode. Choose Debit/Credit Columns or Single Amount. The importer can auto-detect based on your mapping.
  5. Import. Duplicates are skipped automatically. Save your mapping as a Template for one-click reuse.

Full CSV instructions: Import CSV Data.

Planned sources

  • Wallet/banking apps that can export or share statements (CSV preferred).
  • Fintech wallets with standard CSV headers (auto-mapping for common formats).

FAQ

Will I be able to share directly from my wallet?

That’s the goal. We’ll accept shared CSV files and open straight into the importer with mapping pre-filled.

What if my wallet exports OFX/QIF instead of CSV?

CSV will have the smoothest experience. We’ll document any additional formats we add support for. In the meantime, most wallets also offer CSV; if not, a quick converter to CSV will work.

How are currencies handled?

Each transaction stores its currency code. The UI currently displays a single primary currency (USD), with JPY display coming next. See Multi-currency for details.
